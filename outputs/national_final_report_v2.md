# 최종 비교 리포트 v2 (ESS 시뮬레이터 재설계)

생성일시: 2026-05-16 02:26:36

> 본 리포트는 통제된 모델 비교 환경에서의 ESS 시뮬레이션 결과다. 실제 운영값 추정이 아니라 정책 간 상대 비교가 목적이다. 사용 모델은 XGBoost 통합 v2 단일 모델이다.

## 1. 시뮬레이터 재설계 배경

기존 v1 시뮬레이터(`ess_simulation_national.py`)는 다음 6가지 문제를 가졌다. v2는 각각을 아래 방식으로 해결했다.

| # | v1 문제 | v2 해결 |
|---|---------|---------|
| 1 | 단위 버그 잔재 (kWh/MWh 혼재) | 파라미터를 `ess_config_v2.py` 단일 모듈로 통일 |
| 2 | 결정 시점 = 정보 가용 시점 (예측의 시간적 우위 미활용) | 정책 함수가 lookahead 로 다음 시점 예측 참조 |
| 3 | shortage_count 비대칭 + binary 카운팅 | `total/mean/max_shortage_mwh` 로 부족 강도 정량화 |
| 4 | 모든 지역 동일 ESS 파라미터 | 발전량 비중 기반 지역별 차등 분배 |
| 5 | 24시간 고정 수요 | 정성적 시간대별 부하 패턴(정규화) 도입 |
| 6 | 단일 지표(ess_score) 의존 | 자가소비율·자급률·부족 강도 등 다지표 |

## 2. 환경 설정 (통제된 비교 환경)

- 지역별 차등 ESS 파라미터: 17개 지역, 학습 데이터 발전량 비중에 비례 분배
- 전국 합산 ESS 용량: 8500 MWh / 평균 시간당 수요: 850 MWh
- 시스템 RTE: 90% / SOC 운용 범위: 0.20~0.80 (DOD 60%)
- 시간대별 수요: 정성적 한국 부하 곡선 (정규화)
- 4개 정책: naive_baseline / xgb_no_lookahead / xgb_lookahead / oracle
- 사용 모델: XGBoost (national v2, power_diff 포함)

**노이즈 플래그 지역:** ['울산시'] — 학습 데이터상 태양광 발전 비중이 0.1% 미만으로, ESS 용량이 시뮬레이션 노이즈 임계(1 MWh) 수준이다. 가중 평균 결과에는 영향이 거의 없으며, 단순 평균은 17개 전체와 16개(노이즈 지역 제외) 두 기준을 병기한다.

## 4. ESS 시뮬레이션 비교 — 새 지표 기반

### 4-1. 단순 평균 (17개 지역 평등)

### 4-2. 가중 평균 (발전량 비중)

### 4-4. lookahead 도입 효과

`xgb_no_lookahead` → `xgb_lookahead` 자급률 변화량: **-1.4 pt**

> **예상과 반대 결과.** 자급률 순서가 `naive >= lookahead >= oracle` 로 나왔다 (예상: oracle >= lookahead >= naive). 코드 버그가 아니라 시뮬레이터 구조에서 비롯된 구조적 결과다. 시뮬레이터 본체는 매 시점 SOC 범위 안에서 탐욕적으로 최대 충방전하므로, SOC 범위를 가장 넓게(0.20~0.80) 쓰는 naive 가 이미 탐욕 최적이다. `policy_lookahead` 는 충전 상한을 낮추거나 방전 하한을 올려 SOC 범위를 *좁히기만* 하므로 단일 스텝 탐욕 시뮬에서는 naive 를 넘을 수 없다. `naive` 와 `xgb_no_lookahead` 가 완전히 동일한 것도 같은 이유다 (두 정책 모두 SOC 범위 0.20~0.80, 충방전은 실측 기준).

## 5. Sensitivity 분석

예측 정확도(합성 노이즈 수준)를 9단계 x 3 seed = 27점으로 변화시키며 자급률을 측정했다. 곡선: `outputs/ess_v2_sensitivity_curve.png`

| 항목 | 값 |
|------|-----|
| Oracle (noise=0) 자급률 | 79.0% |
| XGBoost 위치 (noise 등가 0.23, nMAE 0.12) | 79.1% |
| Oracle 대비 도달률 | 100.0% |
| 정확도 50% 개선 시 추가 자급률 | -0.1 pt |
| 곡선 기울기 | +0.68 pt / 노이즈 단위 |

곡선 기울기가 **양수**(+0.68 pt/단위)다 — 노이즈가 커질수록(예측이 부정확해질수록) 자급률이 오히려 미세하게 *상승*한다. 4-4의 구조적 결과와 일관된다: `policy_lookahead` 가 손해 구조이므로, 노이즈가 그 lookahead 신호를 망가뜨릴수록 정책이 naive 에 가까워져 결과가 개선된다. 예측 정확도가 ESS 가치로 전환되는 정도는 이 시뮬 구조에서 사실상 0이다.

## 6. 시뮬레이터 한계 (정직한 명시)

본 시뮬은 통제된 모델 비교 환경이며 다음은 후속 과제로 남긴다.

- 실제 KPX 시간대별 수요 데이터 결합
- REC 가중치 / SMP 가격 반영
- 출력제어(curtailment) 시나리오 모델링
- 멀티스텝 MPC 최적화 — 단일 스텝 탐욕 구조에서는 예측 가치가 구조적으로 나타나지 않으므로, 예측의 운영 가치를 보이려면 필수다
- 지역별 ESS 용량 실측 매칭
- `xgb_no_lookahead` 의 기존 v1 시뮬 완전 재현 한계 (새 구조 안의 근사 재현이며 100% 동일하지 않다)
- 울산시 등 노이즈 지역의 태양광 데이터 0 수준 원인 진단 (데이터 진단 단계 과제)

## 7. 핵심 발견 (포트폴리오 시그니처)

- **MAE != ESS 점수** — XGBoost 의 절댓값 정확도(MAE 9.61)와 운영 결정 품질(자급률 74.5%)은 분리된다. 정확도가 좋아도 정책·시뮬 구조가 그 가치를 흡수하지 못하면 운영 지표는 움직이지 않는다.
- **예측 정확도-운영 가치 전환 곡선** — 27점 sensitivity 결과, 곡선 기울기는 +0.68 pt/단위로 양수(역설)다. 정확도 개선의 한계 효용이 0 이하임을 정량적으로 보였다.
- **지역별 모델 성능 → ESS 영향 차등 전파** — 17개 지역에서 발전량 비중에 따라 ESS 파라미터와 자급률이 차등 분포함을 검증했다.
- **구조적 한계의 정직한 진단** — `naive >= lookahead >= oracle` 역전은 단일 스텝 탐욕 시뮬의 구조적 귀결이며, 예측의 운영 가치 실현에는 멀티스텝 MPC 가 필요하다는 결론을 데이터로 뒷받침했다.
